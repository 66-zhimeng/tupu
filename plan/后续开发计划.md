# 研发流程管理系统 — 后续开发计划

> 基于 PRD v2.0 | 创建于 2026-03-14 | 第一期 MVP 已完成

## 一、第一期遗留项（优先完成）

> 这些功能属于 MVP 范围但尚未实现，建议优先处理。

### 1.1 拖拽连线建立依赖
- **需求**：从节点 A 边缘拖出连线到节点 B，自动创建 A→B 依赖关系
- **技术**：AntV G6 `create-edge` 交互插件，拖拽时实时显示虚线预览
- **涉及文件**：`GraphCanvas.tsx`（添加交互插件）、`graphStore.ts`（调用 `addDependency`）
- **预计工时**：4h

### 1.2 父节点展开/收起
- **需求**：双击父节点 → 原地展开子任务网络，子节点有独立位置记忆
- **技术**：G6 子图切换 + combo 或自定义 collapse/expand 行为
- **涉及文件**：`GraphCanvas.tsx`、`graphStore.ts`（管理展开状态）、后端图谱 API（需返回完整子树关系）
- **预计工时**：8h

### 1.3 里程碑位置保存 API
- **需求**：拖拽里程碑节点后持久化坐标
- **技术**：后端 `PUT /api/milestones/{id}` 已支持 position 字段，前端需在 `dragend` 事件中调用
- **涉及文件**：`GraphCanvas.tsx`（区分节点类型调用不同 API）
- **预计工时**：1h

## 二、第二期功能（核心交互增强）

### 2.1 迭代回传机制
- **需求**：A→B 依赖中，B 完成后可激化 A 为"未完成"，记录迭代次数，双方可结束循环
- **后端**：
  - `POST /api/dependencies/{id}/iterate` — 触发迭代回传
  - `POST /api/dependencies/{id}/end-cycle` — 结束循环
  - 更新 `Dependency.iteration_count`，写入 `IterationLog`
- **前端**：
  - 抽屉面板中依赖关系区域增加"迭代回传"和"结束循环"按钮
  - 迭代链路边样式（橙色虚线 + 次数徽标）已在 `GraphCanvas.tsx` 中预设
- **涉及文件**：新增 `api/iterations.py`、修改 `TaskDrawer.tsx`
- **预计工时**：8h

### 2.2 全局搜索与过滤
- **需求**：顶部搜索框按标题/负责人模糊检索，命中后节点居中放大 + 其余降透明度
- **后端**：`GET /api/tasks/search?q=keyword`
- **前端**：
  - `TopBar.tsx` 添加搜索输入框
  - `GraphCanvas.tsx` 实现 `focusNode()` 方法（平移 + 缩放 + 高亮）
  - 过滤器组件（按负责人、按状态、按迭代回传）
- **涉及文件**：修改 `TopBar.tsx`、`GraphCanvas.tsx`、`graphStore.ts`
- **预计工时**：6h

### 2.3 审计日志模块
- **需求**：记录所有关键操作（工时修改、状态变更、依赖增删），在抽屉面板中展示
- **后端**：
  - `AuditLog` 表已创建，需在各 service 中插入日志
  - `GET /api/audit-logs?entity_id=xxx` — 获取指定实体的操作日志
- **前端**：`TaskDrawer.tsx` 添加"操作日志"Tab，时间轴展示
- **涉及文件**：修改 `task_service.py`、`dependency_service.py`、新增 `api/audit_logs.py`、修改 `TaskDrawer.tsx`
- **预计工时**：6h

### 2.4 全量快照与回滚
- **需求**：手动/定时保存全量图谱快照（JSON），支持只读预览和回滚恢复
- **后端**：
  - `POST /api/snapshots` — 创建快照（调用 `GraphService.get_full_graph` 序列化为 JSON）
  - `GET /api/snapshots` — 快照列表
  - `GET /api/snapshots/{id}` — 获取快照数据
  - `POST /api/snapshots/{id}/rollback` — 回滚（清空当前数据，用快照恢复）
  - 定时任务：APScheduler 或 Celery Beat 每周五 23:00 自动创建
- **前端**：
  - 新增快照列表页面/抽屉
  - 快照只读预览模式（物理引擎停止）
  - 回滚确认弹窗 + 自动备份
- **涉及文件**：新增 `services/snapshot_service.py`、`api/snapshots.py`、`SnapshotDrawer.tsx`
- **预计工时**：12h

### 2.5 性能降级策略
- **需求**：单屏节点 > 2000 时自动折叠已完成父任务
- **前端**：`GraphCanvas.tsx` 增加节点计数检测 + 自动 collapse 逻辑
- **预计工时**：3h

## 三、第三期功能（AI 引擎与部署）

### 3.1 AI 增量分析引擎 — 依赖关系推理
- **需求**：新增任务时提取 1-2 度局部子图，LLM 推理前后置关系
- **后端**：
  - `POST /api/ai/infer-dependencies` — 输入任务 ID，返回建议列表
  - 提取局部子图 SQL + 构建 Prompt + 调用 LLM + 解析 JSON
- **前端**：建议连线以绿色虚线预览，逐条/批量确认
- **涉及文件**：新增 `services/ai_service.py`、`api/ai.py`、前端 `AISuggestionOverlay.tsx`
- **预计工时**：12h

### 3.2 AI 任务拆解建议
- **需求**：选中任务 → 点击"AI 拆解" → 根据描述生成子任务方案
- **后端**：`POST /api/ai/decompose-task` — 输入任务 ID，返回子任务标题列表
- **前端**：拆解结果预览列表 + 确认创建
- **涉及文件**：修改 `ai_service.py`、`TaskDrawer.tsx` 添加拆解按钮
- **预计工时**：6h

### 3.3 框选 + AI 智能建联
- **需求**：Shift+拖拽框选多节点 → AI 分析选中节点间的依赖关系
- **前端**：
  - G6 `brush-select` 交互插件（Shift 触发）
  - 框选后底部浮动工具栏
  - 绿色虚线预览 + 逐条/批量确认
- **后端**：复用 `ai/infer-dependencies`，传入选中节点 ID 列表
- **涉及文件**：新增 `SelectionToolbar.tsx`、修改 `GraphCanvas.tsx`
- **预计工时**：8h

### 3.4 AI 限流与熔断
- **需求**：单用户每日限 100 次 AI 调用，管理员可一键关闭 AI
- **后端**：中间件层限流计数 + 配置开关
- **涉及文件**：新增 `middleware/rate_limit.py`
- **预计工时**：3h

### 3.5 MinIO 文件存储集成
- **需求**：任务/里程碑附件的上传、下载、删除
- **后端**：
  - `POST /api/attachments/upload` — 上传到 MinIO
  - `GET /api/attachments/{id}/download` — 生成预签名下载链接
  - `DELETE /api/attachments/{id}` — 删除
- **前端**：`TaskDrawer.tsx` 文件附件区域 + Ant Design Upload 组件
- **涉及文件**：新增 `services/minio_service.py`、`api/attachments.py`、修改 `TaskDrawer.tsx`
- **预计工时**：6h

### 3.6 逾期预警视觉标识
- **需求**：当前时间 > due_date 且未完成时，节点外圈红色虚线边框
- **说明**：前端 `GraphCanvas.tsx` 已实现此逻辑，但需实际运行验证视觉效果
- **预计工时**：1h

### 3.7 Docker Compose 生产部署
- **需求**：一键部署前后端 + 数据库 + MinIO
- **方案**：
  - 前端 Nginx 容器（serve 静态构建产物）
  - 后端 Python 容器（uvicorn 生产模式）
  - PostgreSQL + MinIO 容器
  - 反向代理配置
- **涉及文件**：新增 `Dockerfile`（frontend/backend）、`docker-compose.yml`、`nginx.conf`
- **预计工时**：4h
