## 二、逐模块分析与修正建议

### 1. 系统核心架构（第 3-9 行）

| 问题 | 说明 |
|------|------|
| ❌ 缺少部署方案 | Docker Compose / K8s？单机还是分布式？ |
| ❌ 技术栈过重 | PostgreSQL + Neo4j + MinIO 三套存储对中小团队运维压力大 |
| ⚠️ "百万级节点"存疑 | 研发流程管理系统实际场景是否需要百万级节点？过度设计？ |
| ⚠️ 前端框架未明确 | 只确定了 AntV G6 图引擎，但宿主框架（React/Vue）未定 |

### 2. 任务模型与数据计算（第 11-18 行）

| 问题 | 说明 |
|------|------|
| ⚠️ 递归性能未考虑 | `WITH RECURSIVE` 在深层嵌套（如 10+ 层）下性能如何？是否需要缓存/物化视图？ |
| ❌ 缺少任务状态定义 | 任务有哪些状态？（待开始/进行中/已完成/已暂停/已取消？） |
| ⚠️ 工时单位未统一 | 只提到"小时"，但在面板中是否需要支持"人天"等换算？ |

### 3. 图谱视觉与渲染（第 20-26 行）

| 问题 | 说明 |
|------|------|
| ✅ 视觉映射规则清晰 | 节点大小→工时，饼图→进度，颜色→负责人 |

## 四、🔑 向您提出的关键问题

### 🔴 必须明确的问题（阻塞性）

**Q1. MVP 范围**
> 这份计划包含 10 个需求维度，全部实现工作量至少 3-6 个月（全职团队）。**您计划的第一版（MVP）包含哪些核心功能**？
> 
> 建议 MVP 候选：核心架构 + 任务模型 + 图谱可视化 + 基础面板交互 + 简单权限
> 
> 第二期再加入：AI 引擎、快照、批量导入导出、消息推送

**Q2. 团队规模与开发周期**
> 这个项目是**您一个人开发**，还是有团队协作？预期多久交付第一版？这会直接影响技术选型的复杂度（比如是否真的需要 Neo4j）。

**Q3. Neo4j 是否必要**
> 当前设计中 PostgreSQL + Neo4j 双数据库，运维成本较高。考虑到依赖关系本质上是有向图的边数据，**是否可以先用 PostgreSQL 的邻接表 + `WITH RECURSIVE` 替代 Neo4j**，等数据规模真正需要时再迁移？

**Q4. 用户认证方案**
> 计划中提到了 RBAC 权限，但**完全没有提到用户如何登录系统**。请明确：
> - 自有账号（注册/登录）？
> - 对接已有的企业账号体系（LDAP/AD/飞书/钉钉）？
> - 初期是否可以简化为邀请码或固定账号？

**Q5. 部署方式**
> 系统最终部署在**哪里**？
> - 本地服务器 / 云服务器（阿里云/腾讯云）？
> - Docker Compose 单机部署还是 K8s？
> - 是否需要考虑外网访问？

### 🟡 重要但可稍后确认的问题

**Q6. 前端框架选型**
> AntV G6 作为图引擎已确定，但宿主框架选 **React** 还是 **Vue**？（AntV G6 对 React 生态支持更好，建议 React）

**Q7. "百万级节点" 是否为实际需求**
> 力导向图文档中提到"支持百万级节点"，但研发管理系统的典型场景通常在**几百到几千个任务节点**。请确认这是否是硬性需求，因为这会极大影响前端架构复杂度。

**Q8. AI 与 LLM 选型**
> AI 增量分析引擎使用哪个大语言模型？
> - 云端 API（OpenAI GPT / Claude / 国产如通义千问、DeepSeek）？
> - 是否需要支持本地部署的开源模型？
> - 成本预算大致范围？

**Q9. 多项目隔离**
> 系统是否需要支持**多个独立项目**？不同项目各自拥有独立的任务图谱、里程碑和权限配置？还是初期只需要管理**一个项目**？

**Q10. 实时协作**
> 多人同时查看图谱时，一个人修改了任务进度或新增了依赖关系，**其他在线用户是否需要实时看到变化**（WebSocket 推送）？还是刷新页面即可？
