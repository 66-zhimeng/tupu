## 〇、术语定义

| 术语 | 定义 |
|------|------|
| **力导向图** | 基于物理模拟（弹簧-电荷模型）的节点自动布局算法 |
| **里程碑** | 任务聚类的隐形中心节点，不可手动录入数据 |
| **叶子节点** | 没有子任务的最底层任务节点，唯一允许录入数据的层级 |
| **迭代回传** | 任务链 A→B 中，B 完成后将 A 的状态激化为"未完成"并重新进入工作流的机制 |

## 一、系统核心架构

### 1.1 技术栈

| 层级 | 技术选型 | 说明 |
|------|----------|------|
| **前端框架** | React 18 + TypeScript | AntV G6 对 React 生态原生支持最好 |
| **图渲染引擎** | AntV G6 5.x (WebGL/Canvas) | 力导向图渲染，支持自定义节点、物理引擎 |
| **UI 组件库** | Ant Design 5.x | 表单、面板、抽屉等通用组件 |
| **后端框架** | Python FastAPI | 异步高性能 API，自动生成 OpenAPI 文档 |
| **关系数据库** | PostgreSQL 16 | 存储任务属性、工时、用户信息、审计日志、快照 |
| **图数据库** | PostgreSQL（邻接表） | 初期用 PostgreSQL 的 `WITH RECURSIVE` 实现拓扑查询，替代 Neo4j 降低运维成本 |
| **文件存储** | MinIO（S3 兼容） | 物理附件存储 |
| **AI 接口** | 单一 LLM API（可配置） | 通过环境变量配置 API 地址与密钥，兼容 OpenAI 协议 |

### 1.2 部署方案

- **部署方式**：Docker Compose 单机部署
- **容器编排**：frontend + backend + postgres + minio 四个容器
- **环境**：开发环境本地运行，生产环境部署至云服务器（推荐），支持外网访问
- **反向代理**：Nginx（可选，处理 HTTPS 与域名绑定）

## 二、数据模型

### 2.1 用户模型

- 无 RBAC 权限系统，无登录注册
- 用户以"身份标签"形式存在：任何人创建任务时可自由填写负责人名称
- 系统自动收集所有出现过的负责人名称，作为颜色分配与过滤的基础
- 后续可根据需求扩展为完整的账号体系

### 2.2 项目模型

- 初期支持**单项目**模式，全局共享一个任务图谱
- 项目作为根容器，包含所有里程碑与任务

### 2.3 任务模型

#### 核心字段

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | UUID | 主键 |
| `title` | String | 任务标题 |
| `description` | Text | 任务描述 |
| `parent_id` | UUID (nullable) | 父任务 ID，null = 顶层任务 |
| `milestone_id` | UUID (nullable) | 归属里程碑 |
| `assignee` | String | 负责人名称（自由填写） |
| `estimated_hours` | Float | 预估工时（小时），仅叶子节点手动录入 |
| `status` | Enum | `未完成` / `已完成` / `已取消` |
| `start_date` | Date | 计划开始日期 |
| `due_date` | Date | 计划完成日期 |
| `position_x` | Float | 节点在图谱中的 X 坐标（位置记忆） |
| `position_y` | Float | 节点在图谱中的 Y 坐标（位置记忆） |
| `version` | Integer | 乐观锁版本号 |
| `created_at` | Timestamp | 创建时间 |
| `updated_at` | Timestamp | 更新时间 |

#### 任务状态机

```
                  ┌──────────────┐
                  │   未完成     │ ← 初始状态
                  └──────┬───────┘
                         │
              ┌──────────┼──────────┐
              ↓                     ↓
     ┌────────────────┐   ┌────────────────┐
     │    已完成      │   │    已取消      │
     └────────┬───────┘   └────────────────┘
              │
              ↓ (迭代回传激活)
     ┌────────────────┐
     │   未完成       │  ← 被下游任务回传激化
     └────────────────┘
```

#### 递归汇总规则

- **工时汇总**：父节点工时 = Σ 所有直接子节点工时
- **进度计算**：父节点进度 = Σ(子节点工时 × 子节点完成比例) / 父节点总工时
  - 叶子节点的完成比例：`已完成` = 100%，`未完成` = 0%，`已取消` = 不参与计算

### 2.4 依赖关系模型（邻接表）

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | UUID | 主键 |
| `source_task_id` | UUID | 前置任务 |
| `target_task_id` | UUID | 后置任务 |
| `is_iterative` | Boolean | 是否为迭代回传链路 |
| `iteration_count` | Integer | 已迭代回传次数 |
| `created_at` | Timestamp | 创建时间 |

### 2.5 迭代回传机制

> **核心逻辑**：循环依赖不是错误，而是合法的"迭代回传"业务场景。

- 当 A→B 存在依赖关系时，B 完成后可以将 A 的状态**重新激化为"未完成"**
- 系统记录每次迭代回传的次数和时间
- A 和 B 的负责人均拥有"**结束循环**"按钮，点击后将该迭代链标记为"已完结"，跳转到下一步
- 迭代回传的历史记录完整保留，用于复盘

**迭代回传记录表**：

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | UUID | 主键 |
| `dependency_id` | UUID | 对应的依赖关系 |
| `iteration_round` | Integer | 第几轮迭代 |
| `triggered_by` | String | 触发回传的负责人 |
| `reason` | Text | 回传原因（可选） |
| `created_at` | Timestamp | 触发时间 |

### 2.6 文件附件模型

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | UUID | 主键 |
| `entity_id` | UUID | 关联的实体 ID |
| `entity_type` | Enum | `task` / `milestone` |
| `file_name` | String | 原始文件名 |
| `file_path` | String | MinIO 中的对象路径 |
| `file_size` | BigInt | 文件大小（字节） |
| `uploaded_by` | String | 上传人 |
| `created_at` | Timestamp | 上传时间 |

### 2.7 快照模型

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | UUID | 主键 |
| `name` | String | 快照名称（自定义或自动生成） |
| `snapshot_data` | JSONB | 全量图谱数据（节点 + 边 + 状态 + 坐标） |
| `created_by` | String | 创建人 |
| `created_at` | Timestamp | 创建时间 |

- **全量存储**：每次快照保存完整的图谱状态，不做增量压缩
- **永久保留**：所有快照永不自动删除
- **可回滚**：用户可选择任意快照，将当前图谱数据完全恢复为快照状态

## 三、图谱视觉与渲染规则

### 3.1 节点视觉映射

| 视觉元素 | 映射规则 |
|----------|----------|
| **节点半径** | 严格映射总工时（工时越大，节点越大） |
| **节点填充** | 饼图/扇形：完成部分按负责人颜色着色，未完成部分置灰 |
| **节点外圈** | 默认无边框；逾期（当前时间 > due_date 且未完成）时添加红色虚线边框 |
| **已取消节点** | 整体半透明 + 删除线标记 |

### 3.2 颜色体系

- 采用 **HSL 均匀分布** 配色方案
- 系统自动按负责人首次出现顺序，等间距分配 Hue 值（饱和度 70%，亮度 55%）
- 保证在 20 人以内颜色可清晰区分
- 配色示例：#E74C3C, #3498DB, #2ECC71, #F39C12, #9B59B6, #1ABC9C ...

### 3.3 边缘连线

| 类型 | 视觉样式 |
|------|----------|
| **普通依赖** | 灰色带箭头直线，全局半透明(opacity: 0.3) |
| **迭代回传链路** | 橙色带箭头虚线，显示迭代次数徽标 |
| **高亮状态** | 悬停/选中节点时，关联连线 opacity 提升至 1.0，颜色加深 |

### 3.4 布局规则

- 力导向图布局（Force-Directed）
- 取消全局 Y 轴引力
- 里程碑作为**隐形聚类中心**，归属任务向其聚集
- 节点坐标可手动拖拽调整，调整后的坐标**持久化保存**（位置记忆）

### 3.5 展开与缩放

- 点击父节点：在当前位置展开其子任务网络
- 展开时：物理引擎增加局部斥力，将周围节点推开
- 子节点拥有**独立的位置记忆**：收起后再展开，子节点恢复到上次拖拽后的相对位置
- 收起父节点：子网络折叠回父节点内部，周围节点回弹

### 3.6 性能降级

- 单屏渲染阈值：**2000 节点**
- 接近阈值时自动折叠进度 100% 的父任务节点

## 四、面板交互

### 4.1 通用交互面板

- **触发方式**：点击任意节点 → 弹出右侧抽屉面板
- **面板内容**：
  - 基础信息编辑（标题、描述、负责人、日期）
  - 工时录入（仅叶子节点）
  - 状态切换（未完成 / 已完成 / 已取消）
  - 依赖关系管理（增删依赖线，触发迭代回传）
  - 文件附件管理（上传 / 下载 / 删除）
  - 操作日志（按时间轴倒序展示所有变更历史）

### 4.2 任务创建交互

> 取消 Excel 导入，采用以下交互方式创建任务：

1. **图谱上直接创建**：
   - 在图谱空白区域右键 → 菜单选择"新建任务" → 在点击位置生成新节点
   - 在已有节点上右键 → 菜单选择"新建子任务" → 自动建立父子关系

2. **拖拽连线**：
   - 从节点 A 拖拽连线到节点 B → 建立 A→B 依赖关系
   - 拖拽连线时实时显示连线预览

3. **批量创建（快捷方式）**：
   - 选中一个父节点 → 面板中提供"快速添加子任务"按钮
   - 支持以文本列表形式批量输入子任务标题（每行一个）
   - 系统自动创建并挂载到父节点下

4. **AI 辅助拆解**：
   - 选中一个任务 → 面板中点击"AI 拆解" →  AI 根据任务描述**自动建议子任务拆分方案**
   - 用户确认后批量创建
   - AI 同时推理同一里程碑/父级内的 1-2 度局部拓扑，建议前后置依赖

5. **框选 + AI 智能建联**：
   - **触发方式**：按住 `Shift` + 鼠标拖拽 → 绘制矩形选区 → 框选多个节点
   - **选中态视觉**：被框选的节点外圈显示蓝色高亮边框，其余节点 opacity 降至 30%
   - **操作面板**：框选完成后，底部弹出浮动工具栏，包含：
     - 「AI 智能建联」按钮 → 将选中节点的标题、描述、现有依赖关系打包发送给 LLM
     - 「手动连线」按钮 → 进入连线模式，在选中节点间手动拖拽
     - 「取消选择」按钮
   - **AI 分析范围**：仅在**选中的节点之间**推理依赖关系，不涉及未选中节点
   - **建议预览**：AI 返回后，在选中节点间绘制**绿色虚线箭头**作为建议连线，每条线旁显示依赖原因气泡
   - **确认机制**：
     - 逐条操作：点击绿色虚线 → 弹出选项「✅ 接受 / ❌ 拒绝 / ✏️ 反转方向」
     - 批量操作：工具栏提供「全部接受」/「全部拒绝」按钮
   - **写入时机**：用户确认后才正式写入 `dependencies` 表

## 五、AI 增量分析引擎

### 5.1 AI 接口配置

- 单一 LLM API，通过环境变量配置：
  - `LLM_API_BASE`：API 地址
  - `LLM_API_KEY`：密钥
  - `LLM_MODEL`：模型名称
- 兼容 OpenAI 协议（同时兼容 DeepSeek、通义千问等国产模型）

### 5.2 AI 功能

| 功能 | 触发方式 | 行为 |
|------|----------|------|
| **依赖关系推理** | 新增任务时自动触发 / 手动触发 | 提取同父级/同里程碑的 1-2 度局部子图，推理前后置关系建议 |
| **框选智能建联** | Shift + 框选多节点 → 点击「AI 智能建联」 | 仅在选中节点范围内推理依赖关系，以绿色虚线预览，用户确认后写入 |
| **任务拆解建议** | 手动点击"AI 拆解" | 根据任务描述生成子任务拆分方案 |

### 5.3 AI 限流

- 单用户每日最大请求次数：通过后端配置（默认 100 次/日）
- 管理员可一键关闭 AI 功能，切换为纯人工模式

## 六、审计日志

- 所有关键操作写入 PostgreSQL 日志表
- 记录内容：操作类型、操作人、操作时间、原值、新值
- 覆盖操作：工时修改、状态变更、依赖关系增删、文件增删、迭代回传触发
- 展示：任务面板中"操作日志"Tab，按时间轴倒序

---

## 七、全局检索与过滤

- **全局搜索**：顶部搜索框，支持按任务标题、负责人模糊检索
- **视觉聚焦**：搜索命中后，目标节点居中放大，其余节点 opacity 降至 10%，高亮关联连线
- **动态过滤**：
  - 按负责人过滤
  - 按状态过滤（仅未完成 / 仅已完成）
  - 仅显示含迭代回传的任务
- 被过滤的节点隐藏，物理引擎重新计算布局

---

## 八、快照与回滚

- **手动创建**：用户点击"保存快照"按钮，输入快照名称
- **定时创建**：后端定时任务，每周五 23:00 自动创建快照
- **查看历史**：快照列表页面，展示所有历史快照
- **只读预览**：点击某个快照 → 以只读模式渲染当时的图谱（物理引擎停止）
- **回滚恢复**：点击"回滚到此快照" → 二次确认 → 当前图谱完全恢复为快照状态
  - 回滚前自动创建一个"回滚前自动快照"，防止误操作

---

## 九、并发控制

- 乐观锁：任务表 `version` 字段
- 多人同时编辑同一任务时，后提交者收到"数据已被修改，请刷新"提示
- 前端自动轮询（或后续升级为 WebSocket）保持数据同步
